## 3. 开发任务清单（可执行）

### 3.1 后端任务（P0）

#### T2-BE-01 配额中间件落地

- 目标：分析前执行配额校验，免费用户每日 3 次，Pro/企业不限。
- 接口：
  - 复用：`POST /api/analysis/start`（增加配额拦截）
  - 新增：`GET /api/analysis/quota`
- 涉及文件：
  - `apps/backend/src/routes/analysis.ts`
  - `apps/backend/src/controllers/analysis.controller.ts`
  - `apps/backend/src/services/analysis.service.ts`
  - `apps/backend/src/plugins/redis.ts`
  - `apps/backend/src/middlewares/quota.middleware.ts`（新增）
  - `apps/backend/src/services/quota.service.ts`（新增）
- 关键实现：
  - 统一时区 `Asia/Shanghai` 计算自然日边界。
  - Redis key 建议：`quota:daily:{userId}:{yyyyMMdd}`，TTL 到次日 00:00。
  - 免费用户第 4 次返回 429，业务码如 `QUOTA_EXCEEDED`。
- 验收标准：
  - FREE：同日第 4 次被拦截。
  - 次日 00:00 后自动恢复。
  - PRO/ENTERPRISE 不受限。
- 工时：8h

#### T2-BE-02 历史记录接口

- 目标：提供历史分页与删除能力。
- 接口：
  - 新增：`GET /api/analysis/history?page=1&limit=10`
  - 新增：`DELETE /api/analysis/:analysisId`
- 涉及文件：
  - `apps/backend/src/routes/analysis.ts`
  - `apps/backend/src/controllers/analysis.controller.ts`
  - `apps/backend/src/services/analysis.service.ts`
- 关键实现：
  - 列表按 `createdAt desc`。
  - 返回统一分页结构：`items/page/limit/total/totalPages`。
  - 删除需校验所有权，删除后结果接口返回 404。
- 验收标准：
  - 分页准确。
  - 删除后不可访问。
- 工时：8h

#### T2-BE-03 结果编辑保存接口

- 目标：支持用户编辑单帧提示词并持久化。
- 接口：
  - 新增：`PUT /api/analysis/:analysisId/frames/:frameId/prompt`
- 涉及文件：
  - `apps/backend/src/routes/analysis.ts`
  - `apps/backend/src/controllers/analysis.controller.ts`
  - `apps/backend/src/services/analysis.service.ts`
- 请求示例：
  - `{ "platform": "sora", "language": "zh", "prompt": "...", "negativePrompt": "..." }`
- 验收标准：
  - 保存后刷新页面内容不丢失。
  - 非本人数据不可修改。
- 工时：6h

#### T2-BE-04 单帧重生成接口

- 目标：对单帧重新调用 AI 并更新提示词。
- 接口：
  - 新增：`POST /api/analysis/:analysisId/frames/:frameId/regenerate`
- 涉及文件：
  - `apps/backend/src/routes/analysis.ts`
  - `apps/backend/src/controllers/analysis.controller.ts`
  - `apps/backend/src/services/analysis.service.ts`
  - `apps/backend/src/services/ai-analyzer.service.ts`
  - `apps/backend/src/services/analysis-prompt.service.ts`
- 关键实现：
  - 通过 `thumbUrl` 反解帧图本地路径。
  - 重生成仅影响当前帧，不影响其他帧。
- 验收标准：
  - 单帧重生成成功，UI 可即时看到变化。
- 工时：6h

#### T2-BE-05 导出接口

- 目标：导出 TXT/JSON。
- 接口：
  - 新增：`GET /api/analysis/:analysisId/export?format=txt|json&platform=sora&language=zh`
- 涉及文件：
  - `apps/backend/src/routes/analysis.ts`
  - `apps/backend/src/controllers/analysis.controller.ts`
  - `apps/backend/src/services/analysis.service.ts`
- 关键实现：
  - `Content-Disposition` 返回可下载文件名。
  - TXT 按时间顺序拼接，JSON 保留结构化字段。
- 验收标准：
  - 导出内容准确、格式可读。
- 工时：4h

#### T2-BE-06 分享接口（72h）

- 目标：生成可公开访问的分享链接，并自动失效。
- 接口：
  - 新增：`POST /api/analysis/:analysisId/share`
  - 新增：`GET /api/share/:token`（无需登录）
- 涉及文件：
  - `apps/backend/src/server.ts`
  - `apps/backend/src/routes/share.ts`（新增）
  - `apps/backend/src/controllers/share.controller.ts`（新增）
  - `apps/backend/src/services/share.service.ts`（新增）
  - `apps/backend/src/plugins/redis.ts`
- 关键实现：
  - Redis TTL 固定 72h。
  - 过期返回 410（`SHARE_EXPIRED`）。
  - 输出脱敏，不返回用户隐私信息。
- 验收标准：
  - 72 小时内可访问，超时后返回 410。
- 工时：6h

#### T2-BE-07 提示词多语言与负向提示词

- 目标：补齐 2-2 缺口：语言切换和负向提示词。
- 涉及文件：
  - `apps/backend/src/services/analysis-prompt.service.ts`
  - `apps/backend/src/services/analysis-worker.service.ts`
  - `apps/backend/src/services/analysis.service.ts`
  - `apps/backend/src/types/analysis.ts`
- 关键实现：
  - 每平台输出支持 `zh/en/bilingual` 映射。
  - 为 SD 类平台返回 `negativePrompt`。
  - 结果接口一次性返回多语言数据，避免前端切语言再请求。
- 验收标准：
  - 平台切换与语言切换均可即时更新。
  - SD 平台可见负向提示词区块。
- 工时：12h

#### T2-BE-08 链接下载链路补强

- 目标：消除“失败却 sample_fallback 成功”的误导行为。
- 涉及文件：
  - `apps/backend/src/services/upload-url.service.ts`
  - `apps/backend/src/controllers/upload.controller.ts`
- 关键实现：
  - YouTube/Bilibili 下载失败返回明确错误码。
  - 禁止失败时伪造成功上传结果。
  - 保留解析阶段的清晰错误提示。
- 验收标准：
  - 私密视频、超时、无权限场景均明确报错。
- 工时：8h

## 6. 风险与预案

1. Bilibili 下载链路可能受风控影响：预留 1 天机动，并准备明确失败兜底文案。
2. 分享 token 仅放 Redis 有丢失风险：必要时增加数据库持久化映射。
3. 配额自然日重置易出时区问题：后端统一以 `Asia/Shanghai` 计算并返回 `resetAt`。
