# 视频转提示词网站 · 产品需求文档

**版本：** v1.1 **日期：** 2026-04-12 **状态：** 草稿（已更新）

------

## 1. 项目概述

### 1.1 产品背景

AI 视频生成工具（Sora、Runway、可灵、Pika 等）快速普及，用户需要从参考视频中提炼高质量提示词以复现或创作相似内容。现有工具多为图片提示词提取，专注视频场景的自动化工具仍属空白。

### 1.2 产品定位

面向 AI 视频创作者的在线提示词生成工具。用户上传视频或粘贴链接，平台自动分析画面内容、运镜、风格、节奏，输出适配主流视频生成模型的中英专业提示词。

### 1.3 目标用户

- AI 视频创作者（Sora / Runway / 可灵 / Pika 用户）
- 短视频导演/制片（需要参考视频快速出提示词）
- 广告创意团队（提取品牌视频的视觉风格描述）

------

## 2. 用户系统

### 2.1 注册与登录

- 支持邮箱注册 + 密码登录
- 支持第三方登录：Google、微信（国内用户）
- 登录后才可使用分析功能（未登录用户可浏览首页和示例）

### 2.2 账号功能

| 功能     | 说明                                     |
| -------- | ---------------------------------------- |
| 个人中心 | 查看使用配额、历史记录、账号设置         |
| 配额管理 | 免费版每天 3 次，Pro 无限次，与账号绑定  |
| 历史记录 | 保存最近 50 条分析记录，可重新查看和删除 |
| 偏好设置 | 默认输出平台、默认语言（中/英/双语）     |

### 2.3 安全性

- 密码 bcrypt 加密存储
- JWT Token 鉴权，过期时间 7 天（支持刷新）
- 支持修改密码、注销账号、数据删除请求

------

## 3. 视频输入模块

### 3.1 本地上传

- 支持格式：MP4、MOV、AVI、WebM
- 单文件最大：500MB（免费版限制 100MB）
- 支持拖拽上传，实时显示上传进度

### 3.2 链接导入

支持平台：YouTube、Bilibili、抖音、TikTok、微博视频、小红书视频

**版权处理机制：**

- 用户提交链接时须勾选声明：「我确认该视频为本人拥有或已获授权用于 AI 创作分析」
- 平台展示免责声明：仅提取视频公开画面用于 AI 分析，不存储原始视频内容
- 不支持付费会员专属/加密内容链接的解析
- 保留用户 IP + 操作日志，用于潜在侵权追溯

### 3.3 视频预览

- 上传/解析成功后在线预览
- 显示视频时长、分辨率、帧率等基本信息

------

## 4. 视频分析模块

### 4.1 关键帧提取

- 默认策略：场景切换检测 + 均匀采样（每 5 秒 1 帧）
- 用户可调整采样密度：低（每 10 秒）/ 中（每 5 秒）/ 高（每 2 秒）
- 最多提取 50 帧（超出自动降频）

### 4.2 画面内容分析（每帧）

- 主体识别：人物、动物、物体、建筑等
- 场景识别：室内/室外、自然/城市、时间段（白天/夜晚/黄昏）
- 色调与光影：主色调、对比度风格、光源方向与质感
- 画面风格：写实/动画/电影感/纪录片/广告风等
- 情绪与氛围：紧张、浪漫、压抑、明快等

### 4.3 运镜与动态分析

- 镜头运动：推、拉、摇、移、跟随、固定、手持抖动
- 景别：特写、近景、中景、全景、航拍
- 视频节奏：慢动作、正常、快剪

### 4.4 整体视频分析

- 综合全片输出一段整体风格描述
- 提炼核心视觉关键词（用于提示词 Tag）

------

## 5. 提示词生成模块

### 5.1 目标平台适配

针对各平台的提示词结构和关键词偏好进行专项适配：

| 平台               | 提示词策略                                                  |
| ------------------ | ----------------------------------------------------------- |
| **Sora**           | 自然语言长句，强调动态描述、物理真实感、时序变化            |
| **Runway Gen-3/4** | 镜头指令前置（如 `slow zoom in`），风格词 + 主体 + 动作结构 |
| **Kling 可灵**     | 中文为主，主体动作 + 场景环境 + 镜头语言分段描述            |
| **Pika**           | 简洁英文，突出动效和氛围关键词                              |
| **Wan 万象**       | 中英均可，支持结构化描述格式                                |
| **Hailuo 海螺**    | 中文为主，强调情绪、风格和运镜词汇                          |

### 5.2 输出内容

每次分析输出以下内容：

- **整体提示词**：基于全视频生成，适合整体参考复现
- **分帧提示词**：每个关键帧对应一条提示词，带时间戳标注
- **风格标签组**：提取 10-20 个核心风格关键词，可单独复制使用
- **负向提示词**（Negative Prompt）：适用于 SD 类平台，自动生成排除词

### 5.3 语言输出

- 中文提示词：面向可灵、海螺、万象等中文友好模型
- 英文提示词：面向 Sora、Runway、Pika 等英文模型
- 双语对照：同时输出中英，供用户参考和学习

### 5.4 提示词质量优化

- 专业术语规范化（如统一使用 `cinematic lighting` 而非泛化描述）
- 避免模型敏感词（自动过滤可能触发平台审核的词汇）
- 支持用户手动编辑生成结果后重新优化

## 附录：通用分层视觉素材规范

本规范适用于营销首页、品牌展示页和所有采用 `synex-cinematic-scroll` 方案的整屏滚动场景。目标不是单纯“把图放上去”，而是让设计交付的素材天然适配 `GSAP + ScrollTrigger` 的分镜式叙事、3D 视差、白场过渡和前景遮挡动画。

### A.1 适用范围

- 每个 section 视为一个独立分镜，默认按 `min-height: 100dvh` 设计。
- 每个 section 必须有明确的单一视觉锚点，避免一个画面里同时塞多个主主体。
- 文案、按钮、导航不允许烘焙进位图，必须由前端独立渲染，否则后面做遮罩入场和响应式就是自找麻烦。

### A.2 静态资源目录规范

统一目录：

```text
apps/frontend/public/images/sections/
  s01-hero/
    s01-hero__bg.webp
    s01-hero__mid-main.webp
    s01-hero__fg-rocks.webp
  s02-features/
    s02-features__bg.webp
    s02-features__mid-card-stack.webp
    s02-features__fg-glass.webp
```

命名规则：

- section 目录使用 `sXX-场景名`，例如 `s01-hero`、`s03-focus`
- 文件命名使用 `sectionKey__layer-variant.ext`
- `layer` 只允许 `bg` / `mid` / `fg`
- `variant` 用于区分同层多个素材，例如 `main`、`rocks`、`fog`、`glass`

不推荐继续把所有图都丢在同一个 `backgrounds/` 文件夹里，短期能凑合，后期一多就乱成菜市场。

### A.3 每个 Section 的最低素材要求

| 层级 | 必须 | 格式 | 建议尺寸 | 内容要求 | 动画职责 |
|------|------|------|----------|----------|----------|
| `bg` 背景层 | 必须 | 不透明 `WebP` | `3840x2160` | 完整铺底环境、天空、空间氛围，不带透明通道 | 慢速大位移、轻微缩放、白场/模糊转场承接 |
| `mid` 主体层 | 必须 | 透明 `PNG/WebP` | 长边 `>= 2500` | 主体建筑、核心装置、主卡片、主角色，保留干净 alpha | 作为主叙事锚点，承担主要入场和中速视差 |
| `fg` 前景层 | 必须 | 透明 `PNG/WebP` | 长边 `>= 2500` | 岩石、雾层、玻璃片、枝叶、边框遮挡物等 | 用于遮挡、穿越感、镜头擦过感和叙事加强 |

额外要求：

- 每个 section 至少交付这 3 层，少一层就别硬说自己在做“分层叙事”，那只是单图滚动。
- `fg` 最好不是一整块没意义的装饰边角，而是能形成遮挡关系和空间深度的前景物件。
- 如果同一个 section 需要多个前景元素，建议拆成多个 `fg` 文件，而不是焊死在一张图里。

### A.4 构图与切图规范

- 三层素材必须共享同一透视关系、主光方向和地平线，不允许 `bg` 是左打光、`mid` 变右打光，这种拼法一眼假。
- `bg` 必须完整铺满画布，不能留透明洞，也不能把主体信息堆得太满，避免和 `mid` 抢戏。
- `mid` 主体四周至少预留 8% 到 12% 的安全出血，避免一做缩放或位移就露馅。
- `fg` 需要保留足够大的遮挡面积，建议至少覆盖视口宽度的 12% 到 25%，否则前景存在感太弱。
- 文案安全区要提前让出来，默认桌面端正文区域优先在中央 40% 到 60% 宽度内，移动端要预留居中裁切空间。
- 不要把发光字、数据 UI、按钮、导航、说明文案直接烘焙进图片，这会直接破坏后续的 Text Masking 和交互层级。
- 透明素材边缘必须清理干净，不能有一圈脏灰边、锯齿边、半透明残影，不然后面 blur 一上去就丑得很诚实。

### A.5 动画适配约束

参考 `synex-cinematic-scroll` 的分层逻辑，每层默认承担不同速度和职责：

| 层级 | 推荐速度系数 | 推荐效果 |
|------|--------------|----------|
| `bg` | `0.45` | 缓慢平移、轻微缩放、模糊离场 |
| `mid` | `1.00` | 主体入场、焦点停留、场景切换核心 |
| `fg` | `1.35` | 遮挡掠过、前景压迫、镜头穿越感 |

执行规则：

- 每个分镜只允许一个 `mid` 主焦点承担“主叙事对象”职责。
- 场景切换优先使用 `opacity + blur + scale + yPercent` 组合，不要只做淡入淡出，太像 PPT。
- `fg` 会被优先用于白场过渡前后的遮挡和镜头擦过感，所以前景素材越有轮廓和体积，动画越容易出高级感。
- 如果提供的是雾、玻璃、高光边缘这类素材，前端可以智能叠加模糊、透明度、位移和视差，形成更连贯的叙事动画。

### A.6 导出规范

- 颜色空间统一使用 `sRGB`
- 背景图优先导出为 `WebP`，控制在 `1200KB` 以内
- 透明主体层和前景层优先导出为透明 `WebP`；如边缘质量不稳，再退回 `PNG`
- 单张透明素材建议控制在 `2500KB` 以内，超过就要检查是不是抠图过大或没做区域优化
- 禁止导出设计稿截图、Figma 截屏、带面板边框的半成品，交付必须是可直接上站的最终资产

### A.7 可选增强素材

如果设计资源允许，以下素材会显著提升滚动叙事效果：

- `fog`：雾层或光晕层，适合白场和模糊转场
- `mask`：特殊遮罩形状，适合文字揭示或镜头切换
- `detail`：局部高光碎片、粒子、玻璃反射，用于增强近景层次
- `mobile-crop`：移动端专用裁切版本，避免桌面图在手机上裁得只剩半截

### A.8 交付验收清单

- 每个 section 至少包含 `bg`、`mid`、`fg` 三层
- `bg` 为不透明 `WebP`，`mid`/`fg` 为透明 `PNG/WebP`
- 三层透视、光线、色调一致
- 前景层具有真实遮挡价值，而不是装饰性废图
- 文案安全区预留完成，未把文字烘焙进位图
- 桌面端和移动端裁切都可用
- 文件命名和目录符合规范，可直接被前端接入
